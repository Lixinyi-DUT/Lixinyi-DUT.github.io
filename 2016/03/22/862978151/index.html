它是HTTPS协议，我们怎能不支持它？

  • GitHub
  • HTTPS
  • Hexo
  • SSL
  • 营销软文

扬州计划已全站启用https，看着越来越多的博客都开始启用https，我也跟风部署了SSL证书。本文将详细讲述用Hexo生成部署在GitHub Pages且使用自定义域名重定向的博客如何开启全站https

超文本安全传输协议（HTTPS）也称HTTP Over SSL/TLS,实现原理是在HTTP的基础上再实现SSL(TLS与SSL类似以下不再赘述)那一套的封装，所以HTTPS比起HTTP有一个好，更能保证访客的安全。实现方式是在传输层的所有报文上强制加密。具体来说的技术细节就是获取一个可以被浏览器验证的SSL证书并部署在服务器。当然证书可以由多种方式获得，价格不等。本文提供一种免费且快捷简单的方案（软文）。

一开始我试用了一下cloudflare的方案，似乎有些问题（dns设置变更没有响应），而且它家的cdn加速效果欠佳，于是就放弃了。

然后我要推荐的是kloudsec。原理也是从Let’s Encrypt获取CA再部署在代理服务器上（不使用kloudsec也可以通过Nginx自己搭建自己的代理服务器）。接下来详细说明如何快速开启小绿锁。

注册

https://kloudsec.com/github-pages/new

  • 填写邮箱、设置登陆密码
  • 填写你的GitHub Pages的repo地址、自定义的域名（CNAME指向的地址而非一级域名）

设置DNS

填写完前面两步，将要求你更改域名的DNS设置。增加一条A记录和一条TXT记录。

比如我使用的是DNSPOD，其他如NameCheap应该也是同理。

比如我的二级域名是www.yangzhou301.com那么删除主机记录为www，@和*的dns记录（A和CNAME），再重新按照提示增加两条指向103.203.90.2的A记录，主机记录分别为www和@，另外根据网站给出的值增加一条TXT记录（这一项按照域名生成，每个人的不同）

开启1-Click Encryption

上面的三步完成之后可以进入Dashboard，此时kloudsec将检查你的dns是否设置正确，如果没有问题了，会给你发提醒以及往注册邮箱发邮件。当提示为0 issue时点击左侧的PROTECTION后点击启用SSL Encryption，选择自己的需要部署证书的域名，Automatically redirect和Automatically rewrite assets选项都启用，再开启1-Click Encryption。在dns设置正确的情况下，可以去website settings检查SSL证书部署进度，一般不到一小时就可以在你的网站启用https了。

修改resource

当然这段时间也别闲着，检查一下博客中引用的各种资源的地址，比如md格式中的图片和链接的url地址，将http://改为https://或者//，主题的_config.yml中的头像链接。如果引用了不支持https的资源，那么浏览器上的https是灰色的，无法开启小绿锁。
