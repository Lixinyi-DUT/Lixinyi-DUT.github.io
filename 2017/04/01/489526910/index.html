失踪人口回归，很高兴在新的一年里本站又和大家见面，祝大家春节玉快，豁家欢络，万四如意。

和董先生相见不到3分钟，我就认定他是香港地区的特首了

这样带有硬点意味的话语在恋爱关系中非常常见，但很少会真的在见第一面的时候就脱口而出，一如闯入春晚现场紧紧握住歌唱家的手，这样的勇气非常人所及，但也只能当回想起“我的经历就是……”的时候才敢说这样的话，所以，人们一开始都是怎样确定应该认定哪个人作为自己的伴侣的呢？

相亲场所一经宣传，很快就男男女女人山人海了，这时作为希望得到稳定婚姻的相亲者们竞争不断，如何确定一个算法让所有的相亲者都进入稳定婚姻而避免ntr剧情的出现呢？

2012年经济学诺贝尔奖得主Gale和Shapley提出了Gale-Shapley算法1，证明了在男女数量相同时，总存在一个算法使得所有人进入“稳定”匹配的状态，这个算法也称稳定婚姻/匹配问题（SMP）算法，这个算法经过了几百个教授一致通过，也正是因为这个算法做出的微小工作使得二人获得了诺奖。

问题背景

这个问题不仅限于异性恋婚恋问题，但为了方便讨论我们不妨把问题背景暂时设定于此。

假设有数量相等都为$n$的男女相亲者参与一个大型相亲活动，男性参与者集合为${\bf{M}} = \{ m_1,m_2, \ldots ,m_n\}$，相似地，女性参与者写为集合${\bf{W}} = \{ w_1,w_2, \ldots ,w_n\}$，显然有$\left| {\bf{M}} \right| = \left|{\bf{W}} \right| = n$

对于每个参与这场活动的参与者，对于所有的异性们都会有一个明确的排序。比如3号女士（$w_3$）通过观察和交流对${\bf{M}}$集合的所有男士按照喜爱程度可以给出一个偏序覆盖，如$m_1>m_2>\ldots >m_n$，也就是说如果1号男士和2号男士同时表示有事找大哥，当她需要过河的时候，一定会选择让1号男士来推她。为了方便讨论，同时我们暂时假设这里所有的排序都是严格的，即不存在同时对两位异性好感度完全相同的情况。

稳定婚姻

有人认为SMP模型丝毫不浪漫，甚至说是婚恋中的黑暗森林模型，但笔者我认为这是一派胡言，给SMP泼脏水。虽然把排序这种实际上暗搓搓进行的东西摆到台面上有点残酷，但整个问题的假设还是非常光明磊落的，没有白色相簿2的犹豫不决，没有虎扑步行街的\textcolor[rgb]{0,1,0}{当然是选择原谅她了}，算法运行一次，所有人进入稳定婚姻，买定离手，所有人都接受这个匹配，没有寝取文学出现的余地，相比于现实已经非常干净了。

如何定义这个神奇的稳定婚姻/匹配状态呢？

假设有一对新人$\left( {m,w} \right)$进入了婚姻关系，在怎样的情况下其中会存在一个人，也就是$m$或$w$会出轨？

我们假设这个要出轨的人是$m$，并存在另一对夫妻$\left( m’,w’ \right)$将成为ntr受害者，在异性恋婚恋的背景下，可能与$m$发生不伦恋情的显然是$w’$，但这种不伦不是一定发生，发生需要两个条件：

  • 比起$m$现在的配偶$w$，$m$更喜欢$w’$
  • 比起$w’$现在的配偶$m’$，$w’$更喜欢$m$

概括一下，有妇之夫$m$和有夫之妇$w’$都看自己的原配不如看对方顺眼，他们就会背叛自己原来的家庭，发生婚外情，这时包含$\left( {m,w} \right)$和$\left( m’,w’ \right)$的匹配结果就是不稳定的。

反之，如果婚姻中双方都没有这样的出轨目标，那么这个婚姻就是稳定的。

至此，我们定义稳定匹配为不存在任意一对未配对的异性偏爱对方都胜于自己现在的配偶的匹配方案。

目标

对于$\left| {\bf{M}} \right| = \left|{\bf{W}} \right| = n$设定下男女数量对等的配对方案，显然无论按照什么方法配对总能保证每个人有且只有一个配偶，不会留下“剩男”或者“剩女”，这也是配对本身的要求，问题预设了来相亲的肯定不是不婚主义者而是无论如何必须保证让自己脱单的单身焦虑者，但在这些方案中唯一能采纳的只有保证不会滋生寝取文学的稳定匹配方案。

为了让问题更加生活化，去思考这个问题的我们可以站在这个大型相亲活动的上帝视角来解决问题。这里假设我和读者们都是该相亲活动的举办方，给这$n$男$n$女安排一次集中会面，从而得到每个参与者心目中的对所有异性的好感度排名。

做媒是不包含售后服务的，因此算法只会运行一次将所有的男女生成一对一的配对。假设我们这些媒人举办这样的活动是怀着某种崇高的理想，宁拆十座庙不愿意见到任何一对夫妻分手，那就需要给出一个让$2n$位参与者都愿意乖乖守在自己的婚姻中的配对方案，即使这个方案本身几乎一定不能让参与者满意自己的婚姻。

参与活动的人以什么样的指标评价异性不是我们可以干涉的问题，我们能做的只是根据$2n$位参与者每个人各自对$n$位异性的好感排序硬点出$n$对异性建立关系，并且保证所有配对的人中不会有任何一个人同时具有出轨的野心和能力。

所以总结一下，现在面对这样一个问题，我们要求喂给算法这样的输入，并要求得到这样的输出：

  • 输入：两个元素数量为$n$的两个集合中每个元素对于另一个集合成员的排序
  • 输出：给出$n$对双方分别来自这两个配对的配对，每个元素仅在所有配对中出现一次，不存在任意两个未配对的元素同时认为对方优于现在和他们配对的元素。

Gale-Shapley算法

为了使算法看起来更现实，我们假设男性是主动追求的一方，而女性则对追求者进行考察判断是否接受。对于一位男士$m_i$，按照他对于所有女性的喜好程度可以给出一个向量${\bf{M}}^i = \left[ w_1^i,w_2^i, \ldots ,w_n^i\right]$其中$w_j^i$表示男士$m_i$第$j$喜欢的女士；当然，对应地，女士们心目中也有这样类似的偏好向量。

流程

当然是选择追求她了

正如上文所言，我们采取比较传统的求偶模式，所以这个流程是从男士们的追求开始的，需要主动行动开始追求异性的是还没有找到对象的男士。

除了刚开始的时候，两种情况下男士会单身：

  1. 还没有女士接受过他的追求
  2. 被见异思迁的女朋友甩了

男性的策略是只要还没有配偶就会一直按着他的偏好向量一直追求下去，但不会追求已经追求过的妹子，无论是当时就拒绝了的还是曾经短暂地建立过关系的女性，都不会再去追求了，尽管对于这位男士来说肯定是更喜欢之前追求过的女性。

具体来说对于男士$m_i$，他会根据自己的偏好向量${\bf{M}}^i$由最喜欢的女性依次向后发出追求，一旦追求得到了接受就停止追求，一旦被原来的配偶甩了就继续追求下一位女士，这位女士也就是当前没追求过的女性中他最喜欢的一位。

当然是选择接受他了

女性不会主动地去追求喜欢的可爱的男孩子，只会在接受到男性追求的时候回应接收或者拒绝。

当妹子$w$还没有对象的时候，如果受到一个男性$m$的追求会无条件地接受这样的追求,形成匹配$(m,w)$；如果她已经有了对象，在受到其他男性比如$m’$追求时，会比较对于现在的对象$m$和追求者$m’$的好感度，选择和更喜欢的那位男士建立或维持关系：如果选择了继续和$m$保持关系，则拒绝$m’$的追求；如果选择了和$m’$另起炉灶，那就需要和原来的对象$m$分手，让$m$重回“自由身”。

由于这样被动的接受机制，对于女性来说维护一个不一定要按顺序走完的偏好向量很不方便，不如维护一个好感度函数用来具体比较对于两位男性的好感程度，通俗来说就是依次给异性们打分而不是排序。

匹配器

我们作为主办方当然是要搞一些事情的。参与活动的小哥哥小姐姐们都是非常聪明的人，我们无法干涉也没必要干涉他们的行为模式本身，但我们一句话不说也不好，所以我们要确定一下我们的责任：第一个是读取录入他们的信息（包括编号和偏好向量），第二个是确定整个算法开始的起点和终点，如果说还有一点，就是宣布最后的结果，这对于关注结果的人的命运也是很重要的。

证明

接下来我们将证明这个算法为什么可以保证输出一个稳定匹配的结果，首先我们引入全匹配（perfect match）的概念，当所有的男性和女性都建立了一对一的匹配时，这个匹配被称为全匹配。

首先我们要保证算法的输出结果必须是一个全匹配。回顾一下前文中两性的行为模式，很容易得到

  • 男性在有一个对象后不会继续追求其他女性
  • 女性在找到一个新对象后会和原来的对象分手

上述两个条件保证了无论男女，至多可以拥有一个对象，所以也就不存在脚踏多条船的情况，那么接下来要考虑的就是会不会有人在算法运行结束后依旧单身？

现在我们假设有一个男性追求过所有女性后仍然单身，由于女性的策略是只有在找到下家时才会与原对象分手，单身会直接接受任意追求，所以我们可以推断出所有女性都已经有了对象，但是每个女性都只能有一个对象，那么$n$位女性已配对也可以得到$n$位男性已配对，这与存在单身的男性相矛盾，因此假设不成立，所以输出的配对不会遗漏某个单身人士，必然是一对一的全匹配。至此，输出全匹配证毕。

接下来我们要证明输出的这个匹配还是一个稳定匹配。我们回顾一下前文对稳定匹配的定义，如果不是稳定匹配，考虑一下任意两个最终未互相配对的男女$m$和$w$：

  • 如果$m$追求过$w$：$w$最终没有和$m$配对是因为她和更喜欢的男性配对了，因此比起$m$她更喜欢现在的配偶。
  • 如果$m$没有追求过$w$：那么$m$当然是追求了更喜欢的女性并得到了她的接受所以在没有继续追求$w$，因此比起$w$他更喜欢现在的配偶。

综上所述，未配对的男女之间不可能同时存在喜欢对方超过自己现在的配偶，因此我们得到的输出结果一定是稳定匹配。

复杂度分析

这里$n$为男性或女性的总数量。

时间复杂度

由于整个过程中$n$位男性都会在至多追求$n$位女性，因此总共经历不多于$n^2$次追求，显然可以得到算法的时间复杂度为$o(n^2)$.

空间复杂度

实际上男女参与者需要各维护一个表示偏好的列表，至于辅助空间显然每次迭代都维持在常数级以内，因此整个算法的空间复杂度也为$o(n^2)$.

最佳匹配

实际上稳定匹配往往不止存在一种，我们这种基于男性追求策略的SMP算法输出的仅仅是一个可行的解。通过其他算法，比如女性优先追求可以输出其他可行的稳定匹配结果。

假设有三个应聘者$(A,B,C)$选择参与三个企业$(X,Y,Z)$的招聘4，他们对于对方的排序分别如下

1 2 3
A Y X Z
B Z Y X
C X Z Y
X B A C
Y C B A
Z A C B

这个例子可以有三种两种可行解（稳定匹配）：

  1. 应聘者选择了最喜欢的企业而企业得到了最不中意的应聘者：AY,BZ,CX
  2. 应聘者和企业都选择了彼此眼中“中庸”的选项：AX,BY,CZ
  3. 应聘者去了最不喜欢的企业但企业得到了最中意的应聘者：AZ,BX,CY

显然这三个解都是符合要求的稳定匹配，那么这些稳定匹配结果之间有优劣之分吗？根据实际问题最后应该哪一个稳定匹配作为最优解？

当然对于这个例子来说，第一个解(AY,BZ,CX)是最符合实际的，因为这种工作关系的建立是由应聘者决定优先去哪个企业决定的，把三位应聘者代入主动追求方的角色（男性集合$\bf{W}$），企业则是作为待追求的女士们，运行我们的Gale-Shapley算法，得到的结果就是第一个解。这时显然应聘者们都如愿所偿地进入了最向往的企业，直觉地，这种由应聘者发起追求的策略可以保证输出对于应聘者来说最优的结果。

这时我们所说的“对于某某最优的匹配”还是一种直觉上的说法，那么不妨来定义一下什么是最优的5：

对于男士$m$来说，他最优搭档（optimal partner）$w$是指可以在保证全局最优匹配的情况下他能匹配到的他最喜欢的女士。

对于男士集合$\bf{M}$来说，如果其中每个男士$m$都同时和自己的最优匹配最后建立了关系，那么这个全局的匹配方案就被称为男性最优匹配（male optimal matching）或$\bf{M}$最优匹配。

也就是说如果对于一个输入来说有不止一个稳定匹配结果，那么对于某个$m$来说最优的配偶就是这些结果中与他配对的他最喜欢的女士，对于女性来说也是同理，她能稳定匹配到的最喜欢的男性就是她的最优配偶。同样地，我们可以定义对于某个男性$m$来说他的最差搭档（pessible partner）即所有稳定匹配中他匹配到的最不喜欢的女性，如果有一个稳定匹配使得所有男性同时匹配到了他们各自的最差搭档，那么这个匹配就是男性最差匹配（male pessible matching），当然这些对女性也同理所以这里不再重复说明。

刚才那个招聘的例子中，符合“求职者者主动企业被动”模式的结果恰好是求职者去了“最优企业”而企业却招到了“最差员工”。这种情况并非偶然，而是必然的，主动追求的一方会得到最优匹配，同时被动选择的一方会获得最差匹配。

为了说明主动出击的重要性，还是以上文的角色定位为例，接下来我们将用反证法证明在男性追求女性的模型下得到的稳定匹配，将是对于男性最优（male optimal）的同时也是对于女性最差（female pessible）的。

男性最优

这里将证明上文的算法输出的结果将是男性最优的。

假设得到的结果不是男性最优的，那不妨具体地假设在算法运行中，第一个被自己的最优搭档$w^*$直接拒绝（或者强行分手）的男士为$m$，$w^*$比较了男士$m$和$m^*$后选择的$m^*$,这个事件发生在$t$时刻。

既然$w^*$是$m$的最优搭档，那么肯定存在一个让他们成为配偶的稳定匹配，同时在这里稳定匹配中$m^*$选择了$w^*$以外的女士$w’$，也就是说这个包含这两对男女的稳定匹配可以写为$$S=\{(m,w^*),(m^*,w’),\ldots \}$$
接下来我们开始证明$(m^*,w^*)$是一对会发生婚外情让整个$S$成为不稳定匹配的绿色男女。

首先来看$w^*$既然在我们的算法中比较了男士$m$和$m^*$后选择的$m^*$，那显然$w^*$喜欢$m^*$超过$m$。

另一方面，$t$时刻才发生$\bf{M}$集合中的元素第一次被其最优配偶拒绝（或强行分手，本小节下文的拒绝自带强行分手的情况在内，不再重复说明），那么$[0,t]$的时间内$m^*$还没有被他的最优搭档（记作$w^{**}$）拒绝。$[0,t]$时间段内$m^*$并没有被$w^{**}$拒绝却去追求了$w^*$，说明对于$m^*$来说，$w^{**}$的追求顺序并不在$w^*$之前，也就是说$m^*$对$w^{**}$的喜爱没有超过$w^*$，他对他在$S$中的配偶$w’$的喜爱也一定不高于$w^{**}$，由于$w^{**}$和$w^*$不是同一个人，那么现在可以推出$m^*$喜欢$w^*$超过$m’$。

综上所述，$S$是一个不稳定匹配，也就是说不存在这样的稳定匹配，这与假设矛盾，因此这个假设不成立，基于男性主动追求的算法得到的结果一定是男性最优的。证毕。

女性最差

接下来要证明这个算法的结果对女性是最不利的，在男性最优的稳定匹配中，女性会同时匹配到她们的最差配偶。

假设男性最优的稳定匹配中包含了配对$(m,w)$，即$w$为$m$的最优配偶，如果这个稳定匹配不是女性最差的，那么除此以外必然存在一个这样的稳定匹配$S$使得$w$与某个她更不喜欢的$m’$配对，而$m$和$w$以外的某个女性$w’$匹配，也就是$$S=\{(m,w’),(m’,w),\ldots \}$$

这里我们会着力证明$(m,w)$会发生婚外情破坏$S$的稳定。

因为$w$位$m$的最优配偶，那么显然$m$喜欢$w$是超过$w’$的，构造$S$的时候要求$w$更不喜欢$m’$，那么$w$喜欢$m$也一定是超过$m’$的，那么$(m,w)$互相更喜欢对方而不是在稳定匹配$S$中的配偶，那么$S$就是不稳定的，与假设矛盾，说明不存在这样的稳定匹配。


至此，可以证明算法的输出，也就是男性最优的稳定匹配，一定也同时是对女性最差的。证毕。
