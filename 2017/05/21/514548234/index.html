A: 标题党骗流量而已。作者是废物什么也不会，但不甘心让成功人士独占话语权，因此写这个没有任何技术含量的教程让你感受失败。

Q: 这么没有意义的东西为什么不拿去开知乎live？

A: 因为作者我是失败人士开不了，这还拦不住你们交智商税不如直接给我打钱，，， 好了你已经浪费了20分钟看这些废话问题了，赶紧用剩下的一分钟看看如何制作一个尬聊机器人吧。

设计思路概要

目前废物作者能找到的快速实现qq接受消息和回复的Python方案里面最方便调用的是基于SmartQQ协议的QQbot，用于WebQQ的SmartQQ协议有很多缺陷，比如不能发图片不能圈人，但作为一个简单的聊天机器人的I/O模块已经绰绰有余了。

用QQbot解决了怎么收发消息的问题，那么接下来要解决的就是怎么根据接收的消息发送具体的回复内容，完成这个“思考”的过程。这个话题太大了，现有的解决方案也有很多，比如小冰小娜似乎都提供了接口，直接调用也没太大问题，这里用另一个比较小众的解决方案Wolfram Alpha，优势在于对数学问题的支持比较好，但很突出的实际问题是对中文很不友好。

找到回答

接下来就要考虑怎么用Wolfram Alpha的API实现answer函数了。

首先要去官网申请一个appid，注意非商用的免费appid一个月只能调用API不超过2000次，不过也够用来撸个尬聊机器人玩玩了。

Simple API返回的是查询结果生成的图片概览，Wolfram|Alpha Fast Query Recognizer API返回的是输入能否构成一个有效的查询，Full Results API返回的是完整的包含各种格式资源的结果，无疑Short Answers API 是最适合拿来作为聊天机器人的轮子的。
